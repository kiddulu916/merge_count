Context · 01

The problem

01merge_count has a polished daily ritual but it is one-and-done: a player finishes their four deterministic boards and has zero reason to reopen the app until the next UTC day. There is no variable reward to create dopamine variance, nothing accumulates across days beyond a streak number and a static achievement list, and the friends leaderboard is passive — it shows ranks but never provokes interaction.

02The fix is to layer three missing psychological systems — variable reward, meta-progression, and active async social — onto the existing engine while weaponizing the deterministic shared board the game already generates. Every feature must hold the project's two hard pillars: $0 infrastructure (Supabase free tier + on-device logic, no FCM, no real-time) and a wholesome-but-effective tone (NYT/Duolingo lane, no manipulative timers or guilt).

Commit · 02

Goals

1 Create at least 4 honest daily return moments (staggered content + local notifications) so players reopen the app mid-day, not just once.
2 Add a variable-reward dopamine loop via a deterministic, cheat-proof, $0 Daily Loot Chest and occasional golden tiles.
3 Introduce a soft-currency meta-progression economy that makes the existing cosmetics screen earnable through play, plus a collection/almanac and player level.
4 Convert the passive friends leaderboard into active competition via async duels (deep-link, same-board, auto-compare) and rivalries.
5 Close core UX gaps (first-run tutorial, undo, stats calendar, colorblind-safe tiles) and keep all of it inside the $0 ceiling and wholesome tone.
Signal · 03

Done when…

10 signals
  • 01 Opening the app shows a Daily Loot Chest claimable once per UTC day; the reward amount is derived from the daily seed and is byte-identical for every player on that date; a rewarded ad can double it.
  • 02 At least one bonus board and the loot chest unlock at later times of day, and local notifications (no FCM) fire for midday content, chest-ready, and streak-guard moments.
  • 03 Out-of-moves / deadlock screens display a near-miss line (e.g. '1 merge from tier 512') and, where legal, the rewarded continue.
  • 04 Soft currency is earned from completion, loot, and rewarded ads, and can be spent to unlock cosmetic tile themes in the existing cosmetics screen.
  • 05 A Merge Almanac/collection and a player level visibly accumulate across days and surface on the profile/leaderboard.
  • 06 A player can generate a duel challenge link; a friend opening it plays the identical (date,tier) board and sees an auto-comparison of both scores.
  • 07 A player can set a rival and see a persistent you-vs-them indicator on each board plus a nudge when the rival passes them.
  • 08 The share output is upgraded to a rendered image card (via share_plus) in addition to the emoji block.
  • 09 First-run users get an interactive tutorial; a stats calendar shows past daily results; tiles are distinguishable without color (numerals/patterns).
  • 10 No feature requires paid infrastructure; flutter analyze is clean and flutter test passes including new seed/loot/economy tests.

MVP — must ship ×5
  • Daily Loot Chest — deterministic from daily seed, once/UTC-day, variable soft-currency + rare cosmetic shard, optional rewarded-ad double— Keystone: adds variable reward, seeds the currency economy, and creates a new ad surface at $0.
  • Soft currency primitive + on-device wallet (earn/spend ledger in Hive)— Every economy and cosmetic feature depends on this existing first.
  • Golden tiles — occasional deterministic special drop paying bonus currency on merge— Variable reward inside the core loop, free and cheat-proof.
  • Staggered return moments + local notifications (midday bonus board, chest-ready, streak-guard)— Directly fixes the one-and-done leak; $0 via flutter_local_notifications already in deps.
  • Near-miss framing on out-of-moves / deadlock screens— Weaponizes the 'so close' feeling already produced by the failure states.
Full — target outcome ×8
  • Earned cosmetics economy — wire cosmetics_screen to currency, unlockable tile themes— Turns existing scaffold into a play-driven sink.
  • Merge Almanac / collection + mastery badges with progress bars
  • Player level / XP from cumulative score, surfaced on profile + leaderboard
  • Additional rewarded-ad surfaces (streak freeze, hint, reveal-next-drop)
  • Richer share card — rendered image via share_plus, not just emoji
  • Async duels — deep-link challenge, same (date,tier) board, auto-compare
  • Rivalries — pick a rival, persistent you-vs-them, nudge on pass
  • Polish: first-run tutorial, undo last merge, stats calendar, colorblind-safe tiles
Stretch — if time permits ×1
  • Endless Climb — roguelike anytime mode with meta-progression, gated on Phase 1–3 retention proving out
Out of scope — said no on purpose
  • Referral / invite-to-install rewards — Install attribution is genuinely hard at $0; user deprioritized it.
  • FCM / managed push notifications — Violates the $0 ceiling; local notifications cover the need.
  • Real-time / live multiplayer rivalries — Async by design; live infra costs money and adds ops burden.
  • Hard currency, season pass, IAP store — Monetization stance is ads + earned cosmetics only for now.
  • Aggressive FOMO mechanics (guilt timers, loss-aversion pressure) — Tone locked to wholesome-but-effective.
Future — someday, maybe
  • Light IAP (remove-banner-ads, direct cosmetic purchase) once the earned economy is validated
  • Full F2P economy / season pass if revenue ceiling needs raising
  • Referral rewards if an install-attribution path becomes affordable
  • Seasonal events and seasonal cosmetic rewards
